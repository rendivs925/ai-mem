# ai-mem

**Brain-inspired persistent memory for AI coding agents**

ai-mem provides persistent memory across sessions for any AI coding tool. It captures tool usage, generates semantic summaries, and makes past context available to future sessions — enabling AI agents to maintain knowledge continuity regardless of which tool you use.

## Supported AI Tools

| Tool | Integration |
|------|-------------|
| Claude Code | Native plugin |
| Cursor | Hooks + MCP |
| OpenCode | Native plugin |
| Codex | HTTP API |
| Gemini CLI | HTTP API |
| Any tool | HTTP REST or MCP |

## Quick Start

```bash
# Install ai-mem
/plugin marketplace add thedotmack/ai-mem
/plugin install ai-mem

# Restart your AI tool
# Context from previous sessions will automatically appear
```

## Why ai-mem?

- **Universal** - Works with any AI coding tool via HTTP REST or MCP
- **Brain-inspired** - Uses ACT-R cognitive model for memory prioritization  
- **Privacy-first** - Your data stays local, `<private>` tags excluded
- **Fast** - SQLite with FTS5 full-text search
- **Real-time** - SSE streaming for live memory visualization

## Key Features

- 🧠 **Persistent Memory** - Context survives across sessions
- 🔌 **Universal Integration** - Works with any AI tool via HTTP REST or MCP
- 🧠 **Brain-Inspired** - ACT-R activation model prioritizes important memories
- 📁 **Folder Context Files** - Auto-generated `CLAUDE.md` in project folders
- 🎭 **Mode System** - Switch between workflows (Code, Email Investigation, etc.)
- 🔍 **MCP Search Tools** - Query your project history with natural language
- 🌐 **Web Viewer UI** - Real-time memory stream at http://localhost:37777
- 🔒 **Privacy Control** - Use `<private>` tags to exclude sensitive content
- ⚙️ **Context Configuration** - Fine-grained control over injected context
- 🤖 **Automatic Operation** - No manual intervention required
- 📊 **FTS5 Search** - Fast full-text search across observations

## How It Works

```
┌─────────────────────────────────────────────────────────────┐
│ Session Start → Inject context from last 10 sessions       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ User Prompts → Create session, save user prompts           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ Tool Executions → Capture observations (Read, Write, etc.)  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ Worker Processes → Extract learnings via Claude Agent SDK   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ Session Ends → Generate summary, ready for next session     │
└─────────────────────────────────────────────────────────────┘
```

**Core Components:**
1. **Lifecycle Hooks** - SessionStart, PostToolUse, Summary, Stop
2. **HTTP Worker Service** - REST API on port 37777
3. **SQLite Database** - Sessions, observations, summaries with FTS5
4. **MCP Server** - Model Context Protocol for AI tool integration
5. **HTTP Client SDK** - Easy programmatic access
6. **Web Viewer UI** - Real-time visualization at http://localhost:37777

See [Architecture Overview](architecture/overview) for details.

## System Requirements

- **Bun**: Runtime and process manager (auto-installed if missing)
- **Node.js**: 18.0.0+ (for some integrations)
- **Any AI Coding Tool**: Claude Code, Cursor, OpenCode, Codex, etc.
